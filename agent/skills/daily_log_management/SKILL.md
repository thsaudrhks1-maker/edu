# 프로젝트 문서 및 로그 관리 규칙

## 1. 저장 위치 (Storage Path)
- 모든 프로젝트 관련 문서 및 로그는 루트의 `docs/` 폴더 내에 저장한다.
- **데일리 로그:** `docs/daily_log/YYYY-MM-DD.txt`
- **일반 문서 (매뉴얼 등):** `docs/` 폴더 바로 아래 (예: `docs/Deploy_Manual.txt`)

## 2. 작성 원칙
- 모든 로그 정보는 `docs/daily_log/` 폴더 내에 저장한다.
- 파일명 형식: `YYYY-MM-DD.txt` (Plain Text 형식 유지).
- 하나의 파일에 오전(업무 보고) 및 오후(작업 마무리) 내용을 누적하여 작성한다.

## 3. 문서 포맷 (Plain Text ONLY)
- **마크다운 문법(#, **, [] 등) 절대 금지.**
- 오직 구분선(===, ---), 공백, 들여쓰기, 번호(1.)만 사용한다.
- 메모장(Plain Text)에서 깨지지 않도록 가독성을 최우선으로 한다.

## 4. 작성 내용
- **오전 (업무 보고):** 금일 진행할 업무, 현재 이슈 수정 사항, 리팩토링 및 구조 개선 계획 등.
- **오후 (작업 마무리):** 금일 완료한 작업, 익일 진행 계획, 특이사항 정리.

## 5. 실행 시점
- 에이전트는 사용자가 명시적으로 문서/로그 작성을 지시할 때만 이 규칙을 수행하며, 매 답변마다 자동으로 추가하지 않는다.
